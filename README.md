# Payment Tracking Automation System

A WhatsApp chatbot and automated reporting system for contractor receivables management, built with **n8n**, **WAHA**, **Google Sheets**, and **Gemini AI**.

---

## Overview

This system automates the full lifecycle of contractor payment tracking:

- **WhatsApp chatbot** — query receivables, overdue payments, client profiles, and generate PDF reports via natural language
- **Automated reminders** — escalating WhatsApp notifications sent to the contractor on overdue payments (Day 0, +1, +3, +5, then every 5 days)
- **PDF report generation** — weekly, monthly, yearly, per-client, and general reports with charts and KPI cards
- **Automatic report scheduling** — daily check sends any missing monthly/yearly report via email

---

## Architecture

```
WhatsApp (WAHA)
      │
      ▼
   n8n Workflow
      │
      ├── Gemini AI (NLU — intent + entity extraction)
      │
      ├── Google Sheets (data source — clients & receivables)
      │
      ├── Report API (Flask/Docker — PDF generation)
      │         └── ReportLab + Matplotlib
      │
      ├── WAHA (WhatsApp message sending)
      │
      └── Gmail SMTP (PDF delivery via email)
```

### Three parallel subsystems in one n8n workflow (48 nodes):

| Subsystem | Trigger | Purpose |
|-----------|---------|---------|
| **Chatbot** | Incoming WhatsApp message | Query + on-demand PDF reports |
| **ORK** (Auto Report Check) | Daily cron 09:00 | Send missing monthly/yearly reports |
| **Reminder System** | Hourly cron | Escalating overdue payment reminders |

## Features

### WhatsApp Chatbot
Understands natural language queries in Turkish:

| Query | Response |
|-------|----------|
| Client name / company | Debt status, due date, profile |
| Phone number | Full client details |
| "summary" / "overview" | Total receivables snapshot |
| "overdue payments" | List of all overdue items |
| "due this month" | Payments due in the current month |
| "weekly report" | PDF report → email |
| "monthly report [month]" | Monthly PDF → email |
| "yearly report [year]" | Annual PDF → email |
| "2026 summary" | Annual PDF → email (no keyword needed) |

### Client Profiles
Clients are automatically profiled based on payment history:

| Profile | Condition |
|---------|-----------|
| **Perfect** | ≥95% paid, 0% overdue |
| **Reliable** | ≥80% paid, <20% overdue |
| **Uncertain** | Mixed history |
| **Risky** | Moderate overdue rate |
| **Problematic** | ≥75% overdue rate |

### Reminder Escalation Schedule

| Day | Action |
|-----|--------|
| Due date (Day 0) | First reminder |
| +1 day overdue | Second reminder |
| +3 days overdue | Third reminder |
| +5 days overdue | Fourth reminder |
| Every +5 days after | Recurring until paid |

## PDF Reports

Five report types generated by the Flask `report-api` service:

| Report | Endpoint | Contents |
|--------|----------|----------|
| Weekly | `POST /report/weekly` | Due this week, overdue table, profile distribution |
| Monthly | `POST /report/monthly` | Monthly KPIs, trends, overdue analysis |
| Yearly | `POST /report/yearly` | Annual KPIs, 12-month trend chart, client list |
| Per-Client | `POST /report/client` | Full payment history for one client |
| General | `POST /report/general` | All-time overview across all periods |

## Google Sheets Structure

### `Clients` tab
| Column | Description |
|--------|-------------|
| `client_id` | Unique ID (K001–K050) |
| `phone` | WhatsApp number |
| `full_name` | Client name |
| `company` | Company name |
| `email` | Email address |
| `notes` | Free text notes |
| `active` | Yes/No (soft delete) |

### `Receivables` tab
| Column | Description |
|--------|-------------|
| `client_id` | Foreign key → Clients |
| `receivable_no` | Reference (A001…) |
| `project_name` | Project description |
| `work_description` | Scope of work |
| `total_amount` | Total invoiced |
| `due_date` | Payment due date |
| `status` | Paid / Partial / Unpaid |
| `paid_amount` | Amount received so far |
| `remaining_amount` | Balance outstanding |

### `Report_Log` tab
Tracks automated report delivery (type, period, date, status).

## Setup

### Prerequisites
- Docker + Docker Compose
- n8n instance
- WAHA instance (WhatsApp HTTP API)
- Google Sheets API access
- Gemini API key
- Gmail account with App Password

### 1. Clone and configure
```bash
git clone https://github.com/mustafa3103/payment_tracking_automation_ai.git
cd payment_tracking_automation_ai
cp .env.example .env
# Edit .env with your credentials
```

### 2. Start services
```bash
docker-compose up -d
```

### 3. Import n8n workflow
```bash
# Import workflows/Payment_Tracking.json into your n8n instance
# Update all credentials (Google Sheets, Gmail, WAHA) in n8n
```

### 4. Configure Google Sheets
- Create a Google Sheet with `Clients`, `Receivables`, and `Report_Log` tabs
- Share it with your Google service account
- Add the Sheet ID to your `.env`

### 5. Test the report API
```bash
curl http://localhost:5050/health
# → {"status": "ok"}
```

## Known Limitations

- **WhatsApp file sending** requires WAHA Pro (paid). The free tier only supports text messages. PDF reports are delivered via email in the free tier.
- **Flask dev server** is used in Docker — suitable for low-traffic internal use. Switch to Gunicorn for higher load.
- The system currently runs locally. For production use, deploy to a VPS/cloud server.
